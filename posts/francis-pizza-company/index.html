TREEHOUSE - HTML FOUNDATIONS

Nigel Francis

January 17th 2015

My initial trial with Treehouse had been with their online training. I found the videos, yet good, but a bit cheesy. In time though, I did come across two good books from them. The first: HTML Foundations and later CSS Foundations.

I'll review the former here...


I've been away for over eighteen months from my web-development-study-program (self induced!). Earning a living and off cycling somewhere nigelsbicyclejourneys.com. I once again have a renewed vigor to dig deeper. Too much time on the road thinking!

This started by becoming familiar all over again, with nearly everything it seems. Such a long time away is not conducive with the grey matter... how did I do this coding malarkey in the first place!

One of the key assets to the Treehouse Foundation series is that you'll end up with a usable webpage or two at the end. How you go about presenting it, is entirely up to you.

I found the subject matter in this book (Pizza Co.) pleasant enough to keep, and only changed the Joe's Pizza Co. to Francis's Pizza Co. Yes, not rocket science development, but I'm covering things I supposedly/thought I'd learn't before!

Next something new to me...


WORKING WITH FORMS

THE WORLD WIDE WEB is the single greatest tool ever invented for collecting information. Web forms are responsible for this. We use them every day, whether we are purchasing something online, doing research using our favorite search engine, or logging into our email. Web forms have become an essential part of our everyday browsing experience; in fact, without them, the Internet would be a rather dull place (yes, really).

The executing programme is PHP (Hypertext Preprocessor), which is a server-side scripting language. It's the bones that runs the data, inputted through yourself to the server/s. I'd briefly looked at PHP back nearly two years ago and I still find it puzzling. Well... anything away from HTML and CSS seems to be puzzling!

If you wish to scratch the service of a incredibly well used language, here is a good starting point. There's a few BIG companies that also use PHP: The White House, Yahoo, BBC, Sun, Nike, Novell, Adobe, Rackspace, Symantec, Warner Bros., Greenpeace, Harvard, Fedex, WordPress.com, Forbes, and a billion others! So your in good company.The nice thing is, this can all be implemeneted within your HTML markup.

I went through the process, and as the book guides you through each line of code, it's not so hard. How much I really took in is another thing! However, the template will certainly be a nice source to return to perhaps at a later date.

The book now goes deeper down the rabbit hole with the introduction to:


MICRODATA

Microdata aims to bring structured data models into our web pages by enabling developers to define, use, and share schemas that outline the structure of their data. Think of these schemas as templates of items. They define all the properties that an item could have. A simple schema for a human, for example, would have properties such as height, weight, eye color, hair color and age.

The long and the short of it is... data that you can add, that Google and alike can use to give search engines more information and therefore increase the chances that they'll display this extra information alongside your listing in search results. If building for a client, it's all good stuff to know.

One thing, the more you use microdata, the more complex your code becomes, and this can make it harder to maintain and debug. So be careful.

Further reading on the topic can be found from my favourite information source MDN (Mozilla Developer Network).


ACCESSIBILTY

Accessibility is something I touched on in my first-web-build. Here we go over similar lines.

Building websites that are accessible is crucial to maintaining a web that anyone can use, regardless of whether he or she has a disability.

Making your websites accessible is simply the right thing to do. As a web developer, you should carefully consider your entire audience, not only the typical user, but the full spectrum. What if your user is blind? How will she/he be able to access your content?

Reason 1: It’s the Right Thing to Do

Reason 2: It Leads to Best Practices and Better Code (Google likes that!)

Two ideal tools for this are:

WAVE is a tool to help web developers make their web content more accessible. WAVE can't tell you if your web content is accessible. Only a human can determine true accessibility. But, WAVE can help you evaluate the accessibility of your web content.

NVDA (NonVisual Desktop Access) is a free “screen reader” which enables blind and vision impaired people to use computers. It reads the text on the screen in a computerised voice. You can control what is read to you by moving the cursor to the relevant area of text with a mouse or the arrows on your keyboard.

Both are a must to improve your sites usability for the impaired.


INTRODUCING JAVASCRIPT

JavaScript is a general purpose programming language that is used to create dynamic and interactive websites.

So what are Dynamic Websites? JavaScript enables developers to write programs that can change how a website looks and behaves once it has loaded into the user’s browser. This means, that you could update some of the content in the page without having to reload the page. JavaScript is already in the browser (Chrome, Firefox etc.) you're using to read this.

The chapter touches on:

Variables, Functions, Event Listeners, Loops, Strings, the DOM (Document Object Model) and selecting and interacting with Page Elements.

Also a brief insight to the popular jQuery library for JavaScript.


ADDING VIDEO AND AUDIO

With HTML5 we're able to add video and audio with simple video elements.

ie. <video src="videos/mikethefrog.webm"><video>

But, it's never that easy. Ensuring compatibility with the compatibility of browsers makes it all hard work... again!

My advice would be to use an external source like, Vimeo or YouTube. Keep it simple and pop the embedded code into your site. They're very pliable with the estetic settings too.


STORING DATA

This is another complex subject and one I'm not overly concerned about with my present progress.

You learn how to use the new storage APIs in order to save data on the client-side rather than sending it up to a server to be stored in a database. You'll update the website by adding some JavaScript code that saves users’ contact details when they first submit the bookings form and then automatically populates the Name, Phone, and Email fields for them when they use the form in the future. Again, a useful chapter in the book to come back to when the need arises.


USING GEOLOCATION

The new GeoLocation API is helping to bridge the gap between native applications and web applications.

Here you'll learn how to use the GeoLocation API to request a user’s location. You will be modifying the Locations page so that it highlights the restaurant location that is closest to the user. To do this, you will be using the GeoLocation API and a small JavaScript library that is provided by wiley.com.


USING CANVAS TO CREATE ONLINE ADDS

This is another subject that I went through, but one I'm still unfamiliar with and presently have no need for. Worth it though, just to get an idea. You'll create the green 10% logo that is above. It's not as straightforward as you may think!


ADDITIONAL HTML MARKUP FOR TEXT

The final chapter briefly discusses various elements like em emphasizing text, s strikethrough, ins inserts and more. These are all part of the new HTML5 standard.


SUMMARY

Would I recommend this book ... YES I would. It'll get you up and running with no prior experience at all and if you don't mind the cheesiness of the Treehouse online series, that too would be a good addition. There's a few videos on Treehouse that are also free.

I followed the book to the letter and typed each piece of code as written in the book. This then made for a good refresher.

You can buy the book here TREEHOUSE - HTML FOUNDATIONS

Good luck with your studies and most of all, enjoy it.
