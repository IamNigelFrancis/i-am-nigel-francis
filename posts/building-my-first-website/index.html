BUILDING MY FIRST WEBSITE

Nigel Francis

February 24th 2015

After spending months pretty much studying full time and scratching my head through various training providers and books, I delve into my first-website-build, from the ground up.


FRAMEWORK

I'm starting out with a framework called Bootstrap. Twitter Bootstrap is the most popular front-end framework currently. It's sleek, intuitive, and powerful mobile first front-end framework for faster and easier web development. It utilises HTML, CSS and Javascript.

This gives me a structure and a platform to build from.

I consider my training to date has been fairly robust, so I feel confident I'm not going to make a hash of it! I'd already produced a small website on the topic of online-training-reviews (above) and implemented some jQuery.

First things first though, acquire an understanding of the framework. I found a pleasant layout through Start Bootstrap with a simple business format.

From here, I downloaded the zip-file and extracted to a designated folder. These will contain my CSS and Javascript/jQuery libraries and the HTML index page to get me started.

Don't be too alarmed by all this code, it's only the library that I'll access with code to construct my website. Say for example, I wish to add a navigation bar. This code is written in the index.html page

navbar-inverse navbar-fixed-top role=navigation

With this, I'll access the library and add a default navigation bar. If I wish to change the default, then I'll access my custom.css file with this code in the CSS page.

.navbar-inverse .navbar-nav li a { color: #6495ed; }

Here I'm changing the default colour (#6495ed) to Cornflower Blue in a li (list) a (link)

The key success of Bootstrap is its grid system and how it becomes responsive with all the various devices (tablets, laptops, mobile etc..) we all use today. If your new website is not responsive, then it's viewing potential, especially using a mobile/cellphone is greatly reduced, and with that, the potential for your viewer/customer to go somewhere else with their business. Do you want that… I think not!

Once the folders are in place I can open up the index.html file in my code editor of choice.

Presently, I've been using Microsoft Visual Studios Express 2013 for most of my learning, but found it not to support PHP (scripting language) for my-contact-form page. For this, I use Brackets. I like both editors and have yet to decide on one for future work. Obviously, if I go down the C# route, then Visual Studio it is.

As with anything you build, you might want to start with a wireframe (a page schematic or screen blueprint). I was lazy here and just got straight in, with only a basic drawn sitemap representation. Biting at the bit!

There are many tools out there for just this purpose, but it only becomes another thing to learn. So pad, pencil will suffice for now.

From now on, it's all about what I've learn't and putting it to good practice. To store some of my photos I'm using Imgur. My reason, it's easier to move hosting providers with external links, than it is for photos stored on the hosting providers servers. src="http://i.imgur.com/FbS58Bc.jpg”

The above photo storage has now been deleted. The images are now stored on my hosting providers servers. These are stored in the public_html/posts/building-my-first-website/images/ folder. I'll add further to this in another post.

I've read: Content is King. So with this, I've tried to keep the site simple and not overstated, so the content is not lost in the confusion of functionality.

To change the styling away from the default format, I've added a custom.css file (as mentioned above) so as to override the norm in the CSS library.

Now, this is where Bootstrap comes into its own, with the grid system otherwise known as scaffolding. With this, a page layout can be utilised for a responsive format.


HOMEPAGE

The Home page, been fairly simple with its layout, as I follow the structure of the original Bootstrap grid. Simple things to change: the theme color to #6495ed (hex for Cornflower Blue) and the removal of the well centre.

The addition of the social icons in the top right corner. The Twitter icon code is: fa fa-twitter-square fa-2x

I was keen to remain with the square format for my icons and found these through Font Awesome Icons.

The main photo was sourced, of all places through Google Images… simple! This, then, became my signature photo also for my twitter account @nigelFrancisNET. The reason: bricks - building/construction and the window (microsoft), looking to the future, this also ran with my theme colour.


ABOUT PAGE

The About page needed far more construction, as I start adding the progress bars and my books feature from Goodreads. With this, I needed to change the scaffolding. My progress bars became col-md-8, my training col-md-4 (8 + 4 = 12) and my books col-md-3 (4 x 3 = 12).

But, as I'm learning in this game, once you change or fix one thing, you create ninety nine more problems! Mainly here, the training buttons and book columns would not sit well when I started testing them on different platforms/devices.


PROJECT PAGE

My Project page became the most enjoyable, as I could look back as to how far I'd come. From six months ago just been able to construct a simple HTML page, to my first fully-responsive-website, hosted on a server.

Not as with my cycling blog: nigelsbicyclejourneys.com on the content management system (CMS) that is WordPress.com. With that, the theme is supplied, I just add the content and photos. Hardly any coding is required, compared to my website. Two totally different beasts!

The main photo is downloaded from Unsplash. A free high resolution photographic site. The photos used in letslearncode.co (this blog) header/s, come from this site too.


CONTACT FORM PAGE

The most complex part of the site was my-contact-form page. This, has to be driven by a language. I'd already scratched the surface with PHP (a popular general-purpose scripting language that is especially suited to web development) with some video training from Udemy and The Complete Web Developers Course. This, I now reverted back to and followed the course again, but this time, writing the code into my own designed my contact form.


HOSTING

To host the site, I was fortunate to have a free years subscription to Eco Web Hosting. I acquired this when I purchased The Complete Web Developers Course through Udemy.

Next, was to choose a website domain name. Eventually after running through many names, I stuck with just plain me … nigelfrancis.net. Now (2017) iamnigelfrancis.com

Now, the fun begins, getting my head round cPanel (cPanel is a graphical web-based control panel that helps you quickly and easily manage your website and hosting account).

If you wish for something to initially confuse the heck out of you... cPanel will do it!

Fortunately, The Complete Web Developers and Rob Percival covers the initial intricacies and setup at the beginning of his course, in precise video format.

To link my pages to my hosting provider, I needed a File Transfer Protocol (FTP). This is a standard network protocol used to transfer computer files from one host to another host over a Transmission Control Protocol (TCP) based network.

With today's rich-tech, this can just be a simple add-on to your browser. Here I'm using Firefox's FireFTP. Once I've synced my domain name and password, I'm good to go.

With the contact form written in PHP, I need access to a server to be able to run the program. As you can see from the above image, some of my files are edited with Microsoft Visual Studios Express 2013 and the contact form written with Brackets.

I also set up an images folder, so I could try building with direct-image-links src

<a href="http//nigelfrancis.net/projects/my_postcode_finder/index.php" target="_blank">

<img class="img-responsive img-rounded" src="images/my-postcode-finder-image.png" alt="house on pier over water image">

to photos in the image folder on the host-server-direct. This is more as an exercise, in time, I'll make all the images linked to Imgur or any image-hosting-site.

Some code can still run on a website with errors, but this is unprofessional and can lead to problems debugging (a methodical process of finding and reducing the number of bugs, or defects, in a computer program).

The industry standard is WC3. Here is the validation code for the About page. You can validate your HTML and CSS either through your sites url or direct copy and paste input.

RESPONSIVENESS

Bootstrap may be responsive, but until it's actually tested, you'll see some of it might not work as you wish! I had trouble getting my contact form footer-nav-bar to stay on the bottom, when viewed in portrait mode on all devices.

This was eventually solved by adding a missing </div>. Hours I spent trying to solve this!

MobileTest.me has proven to be a very handy tool indeed. It covers pretty much all device screen sizes that matter.
